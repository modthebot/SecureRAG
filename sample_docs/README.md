# Sample Documents

This directory is for sample documents that can be used to test SecureRAG functionality.

## Guidelines

- **No PII**: Do not include any Personally Identifiable Information (PII)
- **Public Domain**: Only use documents that are publicly available or you have permission to share
- **Generic Examples**: Use generic, non-proprietary documents
- **Test Coverage**: Include various document types (PDF, DOCX, images) to test different parsers

## Suggested Sample Documents

You can add sample documents such as:

- Public technical documentation (e.g., open-source project docs)
- Sample architecture diagrams (anonymized)
- Generic system documentation templates
- Public research papers (with proper attribution)
- Sample markdown files converted to PDF

## Document Types Supported

- **PDF**: `.pdf` files
- **Word Documents**: `.docx` files
- **Images**: `.png`, `.jpg`, `.jpeg` (with OCR support)

## Usage

Place your sample documents in this directory, then:

1. Start SecureRAG
2. Navigate to the Upload page
3. Upload documents from this directory
4. Test the RAG functionality with queries

## Example Queries

Once documents are uploaded, try queries like:

- "What technologies are mentioned in the documents?"
- "Summarize the architecture described"
- "What are the main components?"
- "List all security considerations"

## Note

This directory is excluded from version control by default. If you want to include sample documents in the repository, ensure they are:
- Publicly available
- Free of any proprietary information
- Properly licensed for distribution
